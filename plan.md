設計の流れ（n予備校4章参考）
1. 要求の設定
2. ユースケースの設定
3. 要件定義
4. 用語定義

## 要望:何のためのアプリケーション（サービス）？
- 今日やるレポートを決定する

## ユースケース:どんな時に使う？
- N高等学校に在籍していて一日にやるレポートを決定する　
- 一か月のうち前半だけレポートやりたい場合はそれに沿った予定を作る
- 9/10日を年間の締め切りとして、毎週日曜日を休みとする予定を作る
- 毎月15日を締め切りとして予定を作る
- 偶数月は休みの年間予定

上記のニーズに合うような年間予定が作れる

## 要求定義
なにをしたいか
### 年間予定作成
- 休みの日（レポートをやらない日）をユーザーから取得する
- ユーザーは「締め切り」or「一日の勉強時間」を基準にして予定を作れる
- ユーザーはN予備校のアカウントでログインでき、予定を作る際に未完了のレポート全てを取得する
- 「締め切り」基準の場合は締め切りまでに終わる予定を作る
- 「一日の勉強時間」基準の場合は指定された勉強時間いっぱいに予定を詰め込む
- 「一日の勉強時間」基準で予定を作るとき、締め切り日までに終わらない場合は警告出す
- また一か月のレポート順は「バランス重視」か「特定の教科優先」の二種類を選べる
- 「バランス優先」の場合はバランスよく一日になるべく多くの教科ができるようになる
- 「特定の教科優先」の場合はなるべく一日にする教科の数が少なくなる
- どちらを優先しても、基本的に「NS校が決めた締め切り日を過ぎているレポート」が優先
### 年間予定閲覧
- 今日やるタスクはentrance.htmlのような状態でユーザーに提示する
- 一週間、月間の形で予定を見ることができる
### 年間予定再生成
- 現在の日以前をすべて休日として予定を再生成する
- そのほかの要素に関しては年間予定作成と同じ

## 要件定義
どうやって実装するか
### 年間予定作成
#### レポートの優先順位
締め切り日 > (deadlineOrTime == 'deadline')?():

#### 休みの日（レポートをやらない日）をユーザーから取得する
- 塗り絵のような感じのUI,月、週、曜日、日単位で塗ることができる
- 塗られている場所を↓のようなJSON形式で取得する
- holidayList.length = 休日の数 とする
```json
"holidayList":[
    {"month":1,"day":20},
    {"month":1,"day":21},
    {"month":1,"day":22}
]
```
#### ユーザーは「締め切り」or「一日の勉強時間」を基準にして予定を作れる
```json
"deadlineOrTime":('deadline' or 'time')
```
で返す
#### ユーザーはN予備校のアカウントでログインでき、予定を作る際に未完了のレポート全てを取得する
ログインした状態で、アクセスすればHTMLが取得できるので、そこから分かりそうだけど技術的にできるかどうかがわからない

#### 「締め切り」基準の場合は締め切りまでに終わる予定を作る
月間締め切り日までに行う

#### 「バランス優先」の場合はバランスよく一日になるべく多くの教科ができるようになる
#### 「特定の教科優先」の場合はなるべく一日にする教科の数が少なくなる
#### どちらを優先しても、基本的に「NS校が決めた締め切り日を過ぎているレポート」が優先

#### 「一日の勉強時間」基準の場合は指定された勉強時間いっぱいに予定を詰め込む
#### 「一日の勉強時間」基準で予定を作るとき、締め切り日までに終わらない場合は警告出す
#### また一か月のレポート順は「バランス重視」か「特定の教科優先」の二種類を選べる
```json
"balanceOrSubject":('balance' or 'Subject')
```
で返す

#### 今日やるタスクはentrance.htmlのような状態でユーザーに提示する
#### 一週間、月間の形で予定を見ることができる
#### 現在の日以前をすべて休日として予定を再生成する
#### そのほかの要素に関しては年間予定作成と同じ

## 用語定義
|用語|英語表記|意味|
|:--|:--|:--|
|ユーザー|user|利用者|
|予定|schedules|利用者が持っているすべての予定||
|年間予定|schedule|利用者が持っている年間予定|
|休日|holiday|レポートをやらない日|
|休日リスト|holiday list|レポートをやらない|
|タスク|task|一日にやるレポート一覧|
|レポート|report|レポートのこと|
|単元|unit|レポートが属する単元|
|科目|course|単元・レポートが属する教科 course>unit>report という感じ|
|動画時間|video time|レポートの動画の時間|
|問題数|problem count|問題数|
|いい単語が思いつかない||問題を解くのにかかるであろう時間|
|総時間|total time|レポートを完了するのにかかるであろう時間|
|アカウント設定|user setting|ユーザーの設定|
|締め切り日|deadline list/deadline|6/15,7/15,8/15,9/15,10/15,11/15,12/15|
|レポート最終日|final deadline|レポート最終提出日|
|予定を作る基準|base to create scheduled|締め切り日基準,一日の勉強時間基準|
|締め切り日基準|deadline base|年間予定の決め方|
|一日の勉強時間基準|time base|年間予定の決め方|
|バランス優先|balance preferred|月間予定の決め方。一日になるべく多くの教科のレポートをすることを優先とする|
|教科優先|subjects preferred|月間予定の決め方。一日になるべく少ない教科のレポートをすることを優先とする|
